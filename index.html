Guia rápido para Git/Github

Link do curso.

  • Iniciar um repositório: 
    $ git init
  • Verificar o staus do repositório: 
    $git status
  • Adicionar arquivos ao controle de versão: 
    $ git add "nome_do_arquivo
    ou para todos: 
    $ git add *
    ou 
    $ git add .
  • Criar versões do código com commit 
    $ git commit -m "commit inicial"
    Caso seja necessário, o Git vai pedir pra vc configurar seus dados:
    • Configurar e-mail: 
      $ git commit --global user.email "seuemail@provedor.com"
    • Configurar seu Usernam: 
      $ git config --global user.name "seuUserName"
  • Enviar as alterações registrads no commit. No Github, o repositório "em nuvem" já deve ter sido criado. Para esse guia, o repositório "https://github.com/renzoalves/aprendendo-git"
    1. Adicionando o repositório remoto (sem as aspas): 
      $ git remote add origin "https://github.com/renzoalves/aprendendo-git"
    2. Definir a branch para qual deverá ser enviado os arquivos. Onde "master" é o nome da branch. 
      $ git push --set-upstream origin master
  • Ciclo de atualização de código
    • Adicione novamente os arquivos: 
      $ git add .
    • Verifique se os arquivos que foram atualizados constam na lista: 
      $ git status
    • Agora commite novamente: 
      $ git commit -m "Atualização 2022.03.29"
    • Enviando as atualizações para para o repositório: 
      $ git push
  • Como verificar histórico de atualizações: 
    $ git reflog
    Serão exibidas todas as versões que foram "push-adas".
  • Como navegar entre as versões do seu código:
    • Quando foi rodado o comando "git reflog", o resultado foi esse:

      5d729ec ( HEAD -> master, origin/master ) HEAD@{0}: commit: Atualização 2022.03.29-08:46
      bde8524 HEAD@{1}: commit: commit inicial
      37884a6 HEAD@{2}: commit (initial): commit inicial

      Vamos imaginar que foi cometido um erro e será necessário voltar a uma versão anterior do código. Pra fazer isso, identifique o ID desejado, destacado acima em amarelo.
      $ git reset--hard bde8524

      RESULTADO:

      bde8524 (HEAD -> master) HEAD@{0}: reset: moving to bde8524
      5d729ec (origin/master) HEAD@{1}: commit: commit inicial
      37884a6 (HEAD -> master) HEAD@{2}: commit (initial): commit initial

      Pra voltar pra versão mais atual, é só repetir os passos acima passando o ID da sua última versão.
      A versão mais atual é mostrada pela destacamento: ( origin/master ) .

      Esse procedimento irá alterar o código LOCALMENTE.
  • O que são Branches e como criar uma:

    São caminhos diferentes que você vai seguir durante o seu processo de Desenvolvimento.

    • Ver quais branches estão disponíveis no momento: 
      $ git branch
    • Para criar uma nova branch: (NOTA: "staging" é o nome escolhido para a nova branch) 
      $ git branch staging

      * master
        staging

      A branch marcada com o asterisco(*) é a branch corrente (a branch que você está).

    • Para mudar de branch: 
      $ git checkout staging

        master
      * staging

      Note que agora é a branch 'staging' que está selecionada.

    • Agora vamos fazer o processo pra subir a versão do projeto definida para a branch 'staging'. 
      $ git add .
      $ git commit -m "adicionando nova branch"
      $ git push
      Quando você cria uma branch nova e você quer subir essa branch pro repositório em nuvem, temos de rodar o comando (que o próprio git fornece):
      Note que que é dado o nome da nova branch (atual). 
      $ git push --set-upstream origin staging
